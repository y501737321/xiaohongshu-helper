# Changelog

## v1.1.0 (2026-04-26)

### 搜索优化与精准获客

- **关键词预设**：新增「精准获客」和「泛流量」两组预设词组，教练点击即可批量添加，也可自定义输入
- **广告前置过滤**：标题命中广告关键词（如"接广告"、"课程售价"）的笔记自动跳过，不消耗 AI 额度
- **评论区客户挖掘**：自动扫描评论区，匹配意向词（如"求推荐"、"怎么收费"）的评论者直接记录为潜在客户，零 LLM 开销
- **LLM Prompt 精简**：系统提示词从 ~350 tokens 压缩到 ~100 tokens，输出格式精简，max_tokens 从 4000 降到 2000
- **IP 属地软权重**：不硬过滤外地笔记（避免误杀 VPN / 旅游用户），仅在 AI 评估中作为参考因素
- **广告过滤词/评论意向词可配置**：Settings 中提供预设 + 自定义 tag 输入，教练可灵活调整

### 架构改造（v1.0 → v1.1）

- **去除飞书依赖**：移除 lark-cli，线索改为本地 CSV 存储（按日期分文件，Excel/WPS 直接打开）
- **默认 AI 模型**：从 gpt-4o-mini 切换到 DeepSeek V4 Flash，成本大幅降低
- **扫码登录**：内置浏览器窗口扫码登录小红书，替代终端命令，小白友好
- **自定义保存路径**：线索 CSV 保存位置可由用户自选
- **S 级桌面通知**：极高意向客户弹出系统通知即时提醒教练
- **一键安装依赖**：环境检测横幅中提供 xiaohongshu-skills 一键安装按钮
- **去重持久化**：已处理笔记 ID 和用户 ID 持久化到本地，重启不丢失
- **MCP 服务集成**：使用本地 MCP HTTP 服务替代 Python CLI 调用，更稳定

### CSV 字段

```
账号ID, 昵称, 小红书主页, 笔记ID, 笔记链接, 笔记标题, AI评分, AI摘要, 判断原因, 关键词, 发现时间
```

- 笔记发布者线索：AI评分 = S / A（经 LLM 评估）
- 评论区线索：AI评分 = C-评论（代码匹配，零 LLM 开销）
